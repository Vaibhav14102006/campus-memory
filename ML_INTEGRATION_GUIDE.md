### 2. **Backend API Server** (NEW)
**File:** `backend_server.py`
**ML Endpoints:**

#### `/api/ml/recommend-events` (Lines 117-153)
- **Method:** POST
- **Input:** Student profile (branch, year, gender, skill_level)
- **ML Process:**
  1. Loads trained Random Forest/XGBoost models
  2. Prepares input features (27 features)
  3. Predicts recommendation probability
  4. Predicts satisfaction score
  5. Calculates match scores
  6. Generates recommendations with reasons
- **Output:** Top N recommended events with confidence scores

#### `/api/ml/event-guidance` (Lines 155-177)
- **Method:** POST
- **Input:** Student profile + event name
- **ML Process:**
  1. Filters 100K feedback records for specific event
  2. Finds similar students (same branch/year/skill level)
  3. Analyzes common issues (frequency analysis)
  4. Identifies areas of concern (low ratings)
  5. Identifies strengths (high ratings)
  6. Generates prioritized recommendations
  7. Extracts success tips from high performers
- **Output:** Comprehensive guidance object

#### `/api/ml/predict-event-outcome` (Lines 179-201)
- **Method:** POST
- **Input:** Student profile + single event info
- **ML Process:**
  1. Prepares feature vector
  2. Runs both classification and regression models
  3. Predicts recommendation (Yes/No)
  4. Predicts satisfaction score (1-10)
- **Output:** Prediction with confidence
